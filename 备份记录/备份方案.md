# 聊天记录备份方案

## 一、背景与目标
为确保 clowand 项目资产的安全性与完整性，建立一套自动化、可审计的聊天记录增量备份系统。通过引入"质量审计员"角色，确保备份数据的真实性与正确性。

## 二、日常助手备份执行流程

### 步骤1：检查备份周期
1. 读取 `./备份记录/last_backup_time.txt`
2. 如果文件不存在 → 首次备份，执行步骤3
3. 如果文件存在 → 计算距离上次备份的时间差
   - 不足1小时 → 跳过本次，输出 NO_REPLY
   - 超过1小时 → 执行步骤3

### 步骤2：获取增量聊天记录
1. 使用 sessions_list 获取所有活跃会话
2. 对每个会话使用 sessions_history 获取历史记录
3. 筛选上次备份时间之后的新消息
4. 整理成备份格式

### 步骤3：写入备份文件
文件路径：`./备份记录/chat_backup_YYYYMM.md`（按月份分文件）

格式：
## [YYYY-MM-DD HH:MM] [会话名称/类型]
**用户**: [消息内容摘要]
**智能体**: [回复内容摘要]
---

### 步骤4：更新时间戳
将当前时间写入 `./备份记录/last_backup_time.txt`
格式：`YYYY-MM-DD HH:MM:SS`

### 步骤5：提交审计
完成后，向质量审计员发送审计请求：
@质量审计员 
任务：聊天记录增量备份验证
备份时间：[当前时间]
备份文件：`./备份记录/chat_backup_YYYYMM.md`
时间戳文件：`./备份记录/last_backup_time.txt`
本次备份条数：[X条]
请验证备份完整性和正确性。

## 三、质量审计员验证流程

### 必检项目
| 检查项 | 验证方法 | 通过标准 |
| :--- | :--- | :--- |
| 时间戳更新 | 读取 last_backup_time.txt | 时间戳与报告时间一致 |
| 备份文件存在 | 读取 chat_backup_YYYYMM.md | 文件存在且非空 |
| 备份格式正确 | 检查文件内容 | 包含时间戳、会话名、消息摘要 |
| 增量逻辑正确 | 对比本次备份与上次 | 无重复备份同一时间段 |
| 备份条数合理 | 抽查实际会话记录 | 条数与报告基本一致 |

### 审计报告模板
【审计结果】：✅ 通过 / ❌ 不通过

【检查详情】：
□ 时间戳更新：✅/❌ [实际时间戳：XXX]
□ 备份文件存在：✅/❌ [文件大小：XXX]
□ 备份格式正确：✅/❌ [说明]
□ 增量逻辑正确：✅/❌ [说明]
□ 备份条数合理：✅/❌ [实际：XX条，报告：XX条]

【问题清单】（如有）：
1. [问题描述]

【建议】：[修复建议]

## 四、验收规则（TL补充到提示词）
1. 不接受没有审计员验证的备份报告
2. 必须看到审计员的"✅ 通过"才能确认任务完成
3. 审计员报告"不通过" → 要求日常助手立即修复
4. 连续2次审计不通过 → 下次备份启用双人交叉验证

## 五、首次部署步骤
1. 将方案保存到 `./备份记录/备份方案.md`
2. 初始化检查目录与文件
3. 首次备份并@质量审计员 验证
4. 更新心跳检查清单

## 六、错误处理
- 记录错误到 `backup_error.log`
- 审计失败则修复并重新备份

## 七、文件结构
./备份记录/
├── 备份方案.md              # 本方案文档
├── last_backup_time.txt     # 上次备份时间戳
├── chat_backup_202604.md    # 按月分文件
└── backup_error.log         # 错误日志
